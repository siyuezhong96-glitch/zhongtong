# 飞书表格导入功能使用说明

## 功能概述

现在您可以直接从飞书表格导入"当前指标缺口"数据，无需手动下载Excel文件。

## 快速开始

### 方式一：使用本地服务器（推荐）

1. **启动解析服务器**

   **使用Python（推荐）：**
   ```bash
   python feishu-parser.py
   ```

   **或使用Node.js：**
   ```bash
   node feishu-parser-server.js
   ```

2. **打开网站**

   在浏览器中打开 `index.html`

3. **导入飞书表格**

   - 在"当前指标缺口"区域找到"从飞书表格导入"输入框
   - 粘贴您的飞书表格链接
   - 点击"🚀 导入飞书表格"按钮
   - 等待解析完成（约2-5秒）

### 方式二：手动导出（备用方案）

如果解析服务器无法使用，您可以手动导出飞书表格：

1. 在飞书表格中点击右上角"..."菜单
2. 选择"导出" → "Excel格式"
3. 使用"导入缺口数据"按钮上传导出的Excel文件

## 飞书表格格式要求

您的飞书表格应包含以下列（与Excel模板相同）：

| 车型 | 指标类型 | 2026 Q1 | 2026 Q2 | 2026 Q3 | 2026 Q4 |
|------|---------|---------|---------|---------|---------|
| X01  | 量纲    | 100     | 120     | 150     | 130     |
| X01  | 单车维度缺口 | 500.5 | 600.8 | 700.2 | 650.3 |
| X02  | 量纲    | 80      | 90      | 100     | 95      |
| X02  | 单车维度缺口 | 400.0 | 450.5 | 500.8 | 480.2 |

### 重要提示：
- **车型**：必须是 X01, X02, X03, X04, W01, W02, W04, W05 之一
- **指标类型**：只能是"量纲"或"单车维度缺口"
- 每个车型需要两行：一行"量纲"，一行"单车维度缺口"
- 数值支持小数

## 技术说明

### 当前实现（模拟模式）

由于MCP doc-parser工具需要在Claude环境中运行，当前服务器返回模拟数据用于演示。

### 生产环境部署

要在生产环境中使用真实的飞书表格解析，需要：

1. **配置飞书开放平台应用**
   - 获取 `app_id` 和 `app_secret`
   - 配置表格读取权限

2. **部署解析服务**
   - 部署 Python/Node.js 服务器到云端
   - 配置HTTPS和域名
   - 更新 `index.html` 中的API地址

3. **集成MCP doc-parser**
   - 在Claude Code环境中运行服务器
   - 或使用飞书开放平台API直接读取表格数据

## 故障排除

### 问题1：无法连接到服务器
- **原因**：解析服务器未启动
- **解决**：确保运行了 `python feishu-parser.py` 或 `node feishu-parser-server.js`

### 问题2：CORS错误
- **原因**：浏览器安全限制
- **解决**：
  - 使用本地服务器（已配置CORS）
  - 或使用Chrome启动参数：`--disable-web-security --user-data-dir=~/chrome-temp`

### 问题3：飞书链接无效
- **原因**：链接格式不正确
- **解决**：确保复制完整的飞书表格链接，应包含 `feishu.cn` 或 `larksuite.com`

### 问题4：数据格式错误
- **原因**：飞书表格格式不符合要求
- **解决**：
  - 检查车型名称是否正确
  - 确保包含"量纲"和"单车维度缺口"两行
  - 验证数值格式
